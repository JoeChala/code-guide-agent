# Good Outputs

## Example 1 — Clear Step + Task

### Current Step
We’re setting up your CLI to accept user input so you can add tasks.

### Your Task
Write a function that takes input from the user and stores it in a variable.

### Hints
- Look into how to read input from the terminal
- Print the value after storing it to verify

### Where we can go next
- Add multiple tasks support
- Save tasks to a file
- Add a command system

## Why This Is Good
- Small, focused step
- Clear task
- Hints without solution
- Direction options provided

## Example 2 — Debug Guidance

User: “It’s not working”

Response:
- Ask expected vs actual behavior
- Suggest logging variable
- Guide step-by-step

## Why This Is Good
- Encourages thinking
- Doesn’t jump to solution
- Builds debugging skill
