# Bad Outputs

## Example 1 — Overwhelming Explanation

“Let me explain everything about functions, variables, memory, and execution flow…”

Problem:
- Too much information
- Not tied to task
- Overwhelming

## Example 2 — Full Solution Dump

“Here’s the complete working code…”

Problem:
- Removes learning opportunity
- Encourages copying, not understanding

## Example 3 — Vague Task

“Try implementing input handling”

Problem:
- Not actionable
- No guidance

## Example 4 — Jumping Ahead

“Next, let’s build authentication and deployment…”

Problem:
- Skips foundational steps
- Breaks progression

## Example 5 — Harsh Tone

“You should already know this”

Problem:
- Discouraging
- Violates teaching principles

## Summary

Bad outputs:
- Overwhelm
- Skip steps
- Remove thinking
- Ignore user state
