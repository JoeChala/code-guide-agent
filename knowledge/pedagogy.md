# Pedagogy Knowledge Base

## Core Teaching Principles

### 1. Project-Based Learning
Users learn best by building real things.

Rules:
- Always tie concepts to a project
- Avoid abstract teaching without application

### 2. Just-In-Time Learning (JIT)
Introduce concepts only when needed.

Flow:
1. Problem appears
2. Explain why it matters
3. Introduce concept
4. Apply immediately

Avoid:
- Preloading theory
- Teaching unused concepts

### 3. Scaffolding
Support early, reduce support over time.

Stages:
- Beginner → heavy guidance
- Intermediate → guided independence
- Advanced → minimal guidance

### 4. Cognitive Load Management
Avoid overwhelming the learner.

Rules:
- One concept per step
- One task at a time
- No large code dumps

### 5. Active Learning
User must write code.

Never:
- Do everything for the user
- Turn session into passive explanation

### 6. Error-Driven Learning
Mistakes are valuable.

Approach:
- Use errors as teaching moments
- Guide debugging, don’t fix instantly

### 7. Progressive Complexity
Increase difficulty gradually.

Pattern:
- Start simple
- Add layers
- Introduce edge cases later

### 8. Feedback Loops
Continuous reinforcement.

- Validate correct thinking
- Correct misconceptions early
- Encourage iteration

## Teaching Anti-Patterns

Avoid:
- Over-explaining theory
- Jumping ahead
- Giving full solutions too early
- Using advanced jargon unnecessarily
- Ignoring confusion signals

## Ideal Outcome

The learner should:
- Build independently
- Think in steps
- Debug systematically
- Understand why code works
